# VirtoCommerce UI Kit - Detailed Component Reference

> Enterprise Vue 3 component library for B2B/B2C e-commerce
> Tech: Vue 3 Composition API + TypeScript + TailwindCSS + Headless UI
> Architecture: Atomic Design Pattern (Atoms → Molecules → Organisms)
> Global registration via plugin, all components prefixed with `Vc`

**Important:** After installation, all UI Kit components are globally registered:

### LAYOUT COMPONENTS

**VcLayout** - Smart Two-Column Layout (`client-app/ui-kit/components/atoms/layout/vc-layout.vue`)
- Props:
  - sidebarPosition ("left" | "right", default: "left") - sidebar placement
  - sticky (boolean) - enable smart sticky behavior
  - sidebarAriaLabel (string) - accessibility label for sidebar
- Slots:
  - default (main content area)
  - sidebar (sidebar content)
- Features:
  - **Smart sticky logic**: automatically sticks shorter element (sidebar or content)
  - If sidebar shorter than content → sidebar becomes sticky
  - If content shorter than sidebar → content becomes sticky
  - Disabled on mobile (< md breakpoint)
  - Uses `useSmartSticky` composable with `useElementBounding` to compare heights
  - Responsive layout: vertical on mobile, horizontal on desktop
  - Different sidebar widths by position:
    - Left: w-52 (md+), w-60 (xl+)
    - Right: w-64 (md+), w-72 (xl+)
  - Configurable via CSS variables:
    - --vc-layout-sidebar-offset-top (default: 0px)
    - --vc-layout-sidebar-offset-bottom (default: 20px)
  - Gap between sidebar and content: gap-3 (md+), gap-5 (xl+)
  - Full ARIA support for accessibility
- Use cases:
  - Catalog pages with filters sidebar
  - Product pages with navigation sidebar
  - Documentation with table of contents
  - Account pages with navigation menu

**useSmartSticky** (`composables/useSmartSticky.ts`)

Advanced composable for intelligent sticky positioning. Used internally by `VcLayout`.

**How it works:**

1. **Measures dimensions** using `useElementBounding`:
   - Container top, height
   - Element top, height, width
   - Viewport height, scroll position

2. **Determines affix type** based on:
   - Element height vs viewport height
   - Scroll direction (up/down)
   - Position relative to container
   - Top/bottom spacing requirements

3. **Five positioning modes:**
   - `FIXED_TOP`: Element shorter than viewport, sticks to top
   - `FIXED_BOTTOM`: Element taller, scrolling down, sticks to bottom
   - `ABSOLUTE`: Element taller, scrolling, positioned absolutely
   - `STATIC`: Default position before reaching sticky threshold
   - `STATIC_BOTTOM`: Reached bottom of container

4. **Performance optimizations:**
   - Throttled scroll listener (50ms default)
   - ResizeObserver for reactive size changes
   - Passive scroll events
   - Only calculates when enabled

5. **Returns:**
   - `style`: Reactive CSSProperties to apply to element
   - `isActive`: Whether sticky is currently active
   - `isStuckTop`: Stuck to top of viewport
   - `isStuckBottom`: Stuck to bottom of viewport
   - `update()`: Manually trigger recalculation
   - `destroy()`: Cleanup listeners
